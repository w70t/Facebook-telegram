## 1) تجهيز المشروع
```bash
git clone <repo-url> Facebook-telegram
cd Facebook-telegram
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env        # ثم نعبّيه في الخطوات التالية
```

## 2) ربط حسابك الشخصي في تلغرام
حسابك الشخصي هو الذي **يقرأ** من القناة (لأن بوت تلغرام العادي لا يقدر يقرأ قناة لست أدمن فيها).
يُربط **مرة واحدة فقط** ويتولّد ملف جلسة يُستخدم تلقائياً بعدها.

### أ) احصل على API_ID و API_HASH
1. ادخل <https://my.telegram.org> وسجّل دخول برقم حسابك الشخصي.
2. اضغط **API development tools**.
3. عبّئ أي اسم للتطبيق → ستحصل على:
   - `App api_id`  ← ضعه في `API_ID`
   - `App api_hash` ← ضعه في `API_HASH`

```env
API_ID=1234567
API_HASH=0123456789abcdef0123456789abcdef
```

### ب) سجّل دخول الحساب (أول تشغيل)
شغّل البوت يدوياً أول مرة — سيطلب الرقم ورمز التحقق:
```bash
source venv/bin/activate
python main.py
```
في الترمنال سيظهر:
```
Please enter your phone (or bot token): +9665xxxxxxxx   ← رقم حسابك الشخصي
Please enter the code you received: 12345               ← الرمز يصلك داخل تلغرام
Please enter your password:                             ← فقط لو مفعّل التحقق بخطوتين
```
✅ بعدها يتولّد ملف `user_session.session` ولن يطلب تسجيل دخول مرة أخرى.

> ⚠️ **مهم جداً:**
> - استخدم **رقماً/حساباً ثانوياً** وليس حسابك الرئيسي (أتمتة الحساب مخالفة لشروط تلغرام وقد تُحظر).
> - لازم هذا الحساب يكون **مشتركاً/عضواً في القناة المصدر**.

### ج) حدّد القنوات المصدر
عندك طريقتان:

**الطريقة الأسهل — من داخل البوت (يدعم أكثر من قناة):**
بعد التشغيل، أرسل في قروب المراجعة:
```
/addsource @channel_username     ← إضافة قناة
/addsource -1001234567890        ← أو بالمعرّف الرقمي (قناة خاصة)
/sources                          ← عرض كل القنوات
/delsource @channel_username     ← حذف قناة
```
> شرط: لازم حسابك الشخصي يكون **عضواً في القناة** قبل إضافتها.
> القنوات تُحفظ في `sources.json` وتبقى بعد إعادة التشغيل.

**أو مسبقاً في `.env`** (اختياري، للبداية فقط):
```env
SOURCE_CHANNELS=@channel1,@channel2
```

## 3) بوت المراجعة وقروب الأدمنين

### أ) أنشئ البوت
1. افتح [@BotFather](https://t.me/BotFather) → `/newbot` → اختر اسم ويوزر.
2. ستحصل على **token** ← ضعه في `BOT_TOKEN`.
```env
BOT_TOKEN=123456:ABC-DEF...
```

### ب) أنشئ قروب المراجعة
1. أنشئ قروب جديد في تلغرام.
2. أضف **البوت** و**كل الأدمنين** إلى القروب.
3. ارفع البوت **أدمن** داخل القروب (Manage group → Administrators).

### ج) اعرف الأرقام عبر الأمر `/id`
شغّل البوت، ثم أرسل داخل القروب:
```
/id
```
سيرد عليك:
```
chat id: -1009876543210     ← ضعه في REVIEW_CHAT_ID
your id: 11111111           ← ضعه في ADMIN_IDS (واطلب من كل أدمن يرسل /id)
```
```env
REVIEW_CHAT_ID=-1009876543210
ADMIN_IDS=11111111,22222222
```

## 4) ربط صفحة فيسبوك
فيسبوك يحتاج شيئين: **معرّف الصفحة** `FB_PAGE_ID` و**توكن نشر دائم** `FB_PAGE_TOKEN`.

### أ) أنشئ تطبيق فيسبوك
1. ادخل <https://developers.facebook.com> → **My Apps** → **Create App**.
2. اختر النوع **Business** وأعطه اسماً.

### ب) ولّد التوكن من Graph API Explorer
1. افتح <https://developers.facebook.com/tools/explorer>.
2. اختر تطبيقك من القائمة أعلى اليمين.
3. اضغط **Add a Permission** وفعّل هذه الصلاحيات:
   - `pages_manage_posts`
   - `pages_read_engagement`
   - `pages_show_list`
4. اضغط **Generate Access Token** ووافق على صلاحيات صفحتك.

### ج) احصل على معرّف الصفحة وتوكنها الدائم
التوكن الأول مؤقت (~ساعة). للحصول على توكن الصفحة:
1. في نفس الأداة نفّذ الطلب:
   ```
   GET /me/accounts
   ```
2. في النتيجة، لكل صفحة:
   - `id`           ← ضعه في `FB_PAGE_ID`
   - `access_token` ← ضعه في `FB_PAGE_TOKEN` (توكن صفحة شبه دائم)

```env
FB_PAGE_ID=1234567890
FB_PAGE_TOKEN=EAAB...الطويل
```

> 💡 بما أنها **صفحتك وأنت الأدمن**، تقدر تنشر في **وضع التطوير** بدون مراجعة فيسبوك المعقّدة.
> (النشر لصفحة شخص آخر يتطلب App Review من فيسبوك).

## 5) التشغيل والاختبار

### تأكّد أن الإعدادات صحيحة أولاً
```bash
source venv/bin/activate
python check_setup.py
```
هذا السكربت يختبر اتصال تلغرام وفيسبوك ويخبرك أين الخطأ بالضبط قبل التشغيل الكامل.

### ثم شغّل البوت
```bash
python main.py
```
انشر شيئاً تجريبياً في القناة المصدر → يصلك على قروب المراجعة مع الأزرار:

| الزر | الوظيفة |
|------|---------|
| ✅ نشر | ينشر النص مع الوسائط على فيسبوك |
| 📄 نشر النص فقط | ينشر بدون الفيديو/الصورة |
| ✏️ تعديل النص | ترسل نصاً جديداً ثم يعيد المعاينة |
| ❌ تجاهل | يلغي المنشور |

اضغط **✅ نشر** → يظهر المنشور على صفحتك. 🎉

## 6) التشغيل الدائم على Raspberry Pi
لكي يعمل 24/7 ويعيد التشغيل تلقائياً:
```bash
sudo cp deploy/telegram-fb-bot.service /etc/systemd/system/
sudo nano /etc/systemd/system/telegram-fb-bot.service   # عدّل User والمسارات
sudo systemctl daemon-reload
sudo systemctl enable --now telegram-fb-bot
sudo systemctl status telegram-fb-bot
journalctl -u telegram-fb-bot -f                         # متابعة السجل المباشر
```

## 7) حل المشاكل الشائعة
| المشكلة | السبب / الحل |
|--------|--------------|
| لا يصل المنشور للمراجعة | تأكد أن الحساب الشخصي **عضو في القناة** وأن `SOURCE_CHANNELS` صحيح |
| الأزرار لا تعمل | تأكد أن معرّفك ضمن `ADMIN_IDS` (أرسل `/id`) |
| `/id` لا يرد | البوت غير مضاف للقروب أو ليس أدمن فيه |
| فشل النشر على فيسبوك | التوكن منتهي/ناقص صلاحيات — أعد توليده مع `pages_manage_posts` |
| يطلب تسجيل دخول كل مرة | ملف `*.session` غير محفوظ — لا تحذفه ولا تشغّله كمستخدم مختلف |

> ❗ لا ترفع `.env` ولا ملفات `*.session` إلى GitHub (مستثناة في `.gitignore`).
