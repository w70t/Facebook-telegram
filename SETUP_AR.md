# 📘 دليل الإعداد — كل شيء من داخل تلغرام

لا حاجة لتعديل أي ملفات. فقط أمر واحد لمرة واحدة، ثم كل شيء من البوت بالأزرار.

---

## 1) التثبيت على Raspberry Pi
```bash
git clone https://github.com/w70t/Facebook-telegram.git
cd Facebook-telegram
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
```

## 2) الإعداد الأولي (مرة واحدة فقط)
هذه القيم الثلاث لازمة لتشغيل البوت نفسه:
```bash
python setup.py
```
سيسألك:
- **API_ID** و **API_HASH** ← من <https://my.telegram.org> ← API development tools
- **BOT_TOKEN** ← من [@BotFather](https://t.me/BotFather)

تُحفظ في `settings.json`. ما راح تحتاج تكررها.

## 3) شغّل البوت
```bash
python main.py
```
في السجل بيظهر سطر مثل:
```
لا يوجد مالك بعد. أرسل للبوت في تلغرام:  /claim 123456
```

## 4) صيّر نفسك المالك
افتح بوتك في تلغرام وأرسل:
```
/claim 123456
```
✅ صرت المالك والأدمن. أرسل `/panel` وتظهر **لوحة التحكم**.

---

## 5) أكمل الإعداد من اللوحة `/panel` (بالترتيب)

### 🔐 تسجيل دخول الحساب الشخصي
1. اضغط الزر → أرسل رقم هاتفك:
   - مع رمز الدولة: `+9665xxxxxxxx`
   - أو بدونه، بشرط تضبط **🌍 رمز الدولة الافتراضي** أولاً (مثل `966`).
2. يوصلك رمز داخل تلغرام. **أرسله مع فواصل** حتى لا يُلغى تلقائياً:
   ```
   1 2 3 4 5
   ```
3. لو حسابك محمي بتحقق بخطوتين، بيطلب كلمة المرور.

✅ بعدها الحساب مسجّل وملف الجلسة محفوظ.
> استخدم **رقماً ثانوياً** ولازم يكون **عضواً في القنوات** التي ستضيفها.

### 📘 إعداد فيسبوك
اضغط الزر ثم أرسل:
1. **معرّف الصفحة** (FB_PAGE_ID)
2. **توكن الصفحة** (FB_PAGE_TOKEN)

من أين تجيبهم؟ من <https://developers.facebook.com/tools/explorer>:
- فعّل الصلاحيات: `pages_manage_posts` و `pages_read_engagement` و `pages_show_list`.
- نفّذ `GET /me/accounts` → لكل صفحة `id` (المعرّف) و `access_token` (التوكن).

### 📍 تعيين قروب المراجعة
- أنشئ قروب، أضف الأدمنين والبوت، وارفع البوت أدمن.
- من داخل القروب اضغط الزر (أو أرسل `/panel` ثم 📍) → تصير هذه المحادثة مكان المراجعة.

### 📡 القنوات المصدر
- اضغط **➕ إضافة قناة** ثم أرسل `@يوزر_القناة` أو رابطها أو معرّفها الرقمي.
- تقدر تضيف **أكثر من قناة**. **➖ حذف** لإزالتها.
> شرط: حسابك الشخصي **عضو في القناة**.

### 🐦 حسابات X (تويتر) — اختياري
لنسخ التغريدات مجاناً (طريقة غير رسمية عبر `twikit`):
1. اضغط **🐦 حسابات دخول X** → **➕ إضافة حساب دخول** ثم أرسل بالترتيب:
   - اسم المستخدم (بدون @)
   - البريد الإلكتروني (أو `-` لتخطّيه)
   - كلمة المرور (تُحذف رسالتها تلقائياً للخصوصية)
2. بعد نجاح الدخول، اضغط **🐦 حسابات X المتابَعة** → **➕ إضافة حساب** وأرسل `@اسم_الحساب`.
3. يفحص الحسابات كل ~دقيقتين **بفاصل زمني عشوائي** (لتقليل الكشف الآلي)، وأي تغريدة جديدة تدخل نفس خط المراجعة → الأزرار → فيسبوك.

**لو انحظر حساب الدخول:**
- أضف حساب X آخر من **🐦 حسابات دخول X ← ➕ إضافة** — والبوت **يبدّل إليه تلقائياً** وينبّهك، بدون ما تخسر قائمة الحسابات المتابَعة.
- الرموز: ⭐ الحساب النشط، 🚫 محظور/فشل. وزر **♻️ إعادة تفعيل المحظورة** لو رجع أحدها يشتغل.

> ⚠️ **مهم:** استخدم **حسابات X ثانوية** — الطريقة مخالفة لشروط X وقد تُحظر الحسابات.
> لو طلب تأكيداً أمنياً عند الدخول، سجّل دخول الحساب من المتصفح مرة ثم أعد المحاولة.

### 👤 الأدمنون (للمالك)
أضف أدمنين آخرين بمعرّفهم الرقمي (يعرفونه بأمر `/id`).

## 6) الاستخدام
أي منشور جديد في القناة → يوصل لقروب المراجعة مع الأزرار:

| الزر | الوظيفة |
|------|---------|
| ✅ نشر | ينشر النص مع الوسائط |
| 📄 نشر النص فقط | بدون الفيديو/الصورة |
| ✏️ تعديل النص | ترسل نصاً جديداً ثم يعيد المعاينة |
| ❌ تجاهل | يلغي المنشور |

---

## 7) التشغيل الدائم + التحديث الذاتي
```bash
sudo cp deploy/telegram-fb-bot.service /etc/systemd/system/
sudo nano /etc/systemd/system/telegram-fb-bot.service   # عدّل User والمسارات
sudo systemctl daemon-reload
sudo systemctl enable --now telegram-fb-bot
journalctl -u telegram-fb-bot -f
```
لتحديث الكود لاحقاً: اضغط زر **🔄 تحديث من GitHub** داخل البوت — يسحب آخر نسخة ويعيد تشغيل نفسه تلقائياً.

---

## حل المشاكل
| المشكلة | الحل |
|--------|------|
| `/claim` لا يعمل | خذ الرمز من سجل التشغيل (`journalctl -u telegram-fb-bot`) |
| الرمز يُلغى عند تسجيل الدخول | أرسله مع فواصل: `1 2 3 4 5` |
| رقم بدون رمز دولة | اضبط 🌍 رمز الدولة الافتراضي أو أرسل الرقم كـ `+...` |
| لا يصل منشور للمراجعة | تأكد أن الحساب عضو في القناة وأنك عيّنت 📍 قروب المراجعة |
| فشل النشر على فيسبوك | التوكن ناقص صلاحيات — أعد توليده مع `pages_manage_posts` |

> ❗ لا ترفع `settings.json` ولا ملفات `*.session` إلى GitHub (مستثناة في `.gitignore`).
